Why(or Why Not) to use a CMS to Build Your Site



A Content Management System (CMS) makes it easy for people with little to no web development skills create a website. Some examples of CMSs are WordPress, Drupal and many more. Personally, I am not a fan but I can recognize the benefits it can have. Below are a few pros and cons. I tried to be as impartial as I could and below that I will explain how I build my site and my reasoning behind it.

Pros:

  • Easy to Get Started

    • Although it may not perform, or look as well, as it could, you can get away with creating a WordPress blog without writing any code yourself if you wanted to.

  • Free

    • Free is always good so it’s fine if you just want a personal website or blog but there are some caveats that you should know about if you are using it in a professional setting.

  • Easy to Edit/Add New Content

    • With a CMS all you need to do is log on to the website, click new post, type the content, click publish and you’re done. One of the key reasons to use a site with a CMS is if you have a site that updates its content frequently, like a blog.

  • Get the Website Out The Door Faster

    • Most of the work is already done for you. All you need to do is buy a theme, or use one of the free ones, and put in your content and you’re good to go.

  • Many CMS Sites Have a Mobile App

    • That allows you to edit your site’s content from anywhere.

Cons:

  • It's Too Easy

    • Sure you don’t need to code to create your site but if you want to customize it more than the dashboard lets you do, you need to code. Also, it can be more difficult to develop for a site using a CMS if you don’t know what you’re doing.

  • Anything Free Has a Catch...

    • With it being free, there is a lack of control. An example of this is the domain name. For example, if you use the free package from WordPress your site will have a domain name like sitename.wordpress.com. That’s just one example of extras you can buy from a site using a CMS.

  • Security Issues

    • I am a proponent of open source software but with CMSs being built on open source frameworks, that can lead to many flaws in security. Plugins can also be vulnerable to attack, which makes your site vulnerable as well.

Like I said in the beginning of this post, I am not particularly a fan of CMSs. This site is hosted using Amazon S3 which is powered by Amazon Web Services. I might have gone with a CMS if I was updating my content more frequently but Amazon S3 is perfect for a static site. It gives me a lot more flexibility and freedom to do what I want with it and I’m just now scratching the surface.
